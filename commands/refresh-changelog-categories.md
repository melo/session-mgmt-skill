Generate or refresh the `.claude/changelog-categories.yml` file by analyzing the project's directory structure.

No arguments required.

## Steps

### 1. Analyze project structure

Explore the project's top-level and second-level directories to understand the codebase layout:
```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
```

Look at the directory tree, focusing on:
- Source code directories (`src/`, `lib/`, `app/`, etc.)
- Test directories (`tests/`, `test/`, `spec/`, etc.)
- Configuration and infrastructure (`deploy/`, `.github/`, `docker/`, etc.)
- Documentation (`docs/`, `README*`, etc.)
- Database (`migrations/`, `alembic/`, `prisma/`, etc.)
- Frontend assets (`static/`, `public/`, `assets/`, `templates/`, etc.)

### 2. Read existing config (if any)

Check if `.claude/changelog-categories.yml` already exists:
```bash
cat "$REPO_ROOT/.claude/changelog-categories.yml" 2>/dev/null
```

If it exists, use it as a starting point — preserve any manual customizations the user has made.

### 3. Infer categories

Group directories into meaningful categories. Common patterns:

- **APIs / Backend**: `src/api/`, `src/routes/`, `src/services/`, `src/repositories/`
- **Frontend**: `src/components/`, `src/pages/`, `templates/`, `static/`
- **Database**: `migrations/`, `alembic/`, `prisma/`
- **Infrastructure**: `deploy/`, `.github/`, `docker/`, `Dockerfile*`
- **Tests**: `tests/`, `test/`, `spec/`
- **Documentation**: `docs/`, `schemas/`
- **Configuration**: `scripts/`, config files at root

Adapt category names to the specific project. Use names that make sense for the codebase, not generic labels.

### 4. Write the file

```bash
mkdir -p "$REPO_ROOT/.claude"
```

Write `.claude/changelog-categories.yml` with the inferred mappings. Format:

```yaml
# Auto-generated by session-mgmt skill — edit freely.
# Maps directory prefixes to changelog categories for /daily-changes.
CategoryName:
  - path/prefix/
  - another/path/
```

### 5. Report

Show the user the generated file and ask if they want to adjust anything before committing.
